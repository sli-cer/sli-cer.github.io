Hexo+Github搭建个人博客

1.前言

1.1技术及优势

本文的目的是指引有需求的同学搭建起自己的个人博客。先介绍一下本文用到的技术，然后顺便一提使用这些技术的好处。

Github：提供云端的数据仓库以存储博客内容，配合Git实现版本管理，pages服务可在线解析网站

Git：版本管理工具，实现本地数据与云端同步，方便安全

Hexo：比较成熟的博客框架，提供博客的所有基本功能，方便易用，节省大量时间

Markdown：写博客所用的语言，使用Hexo也可解析其它语言，如EJS或Pug，本博使用Markdown

1.2工具及环境

在搭建博客之前要先准备一些工具和环境：

  1. 本地安装Git客户端
  2. 注册Github账号
  3. 安装node.js
  4. 安装Hexo（在安装node.js之后一条命令搞定）

本文使用的环境：

  • Win10
  • node@14.8.0
  • Git@2.26.2
  • Hexo@5.1.0

1.3初始化

环境安装完成后，使用cmd，cd到选好的目录（这里就是博客文件的存储目录）下，执行如下命令：

hexo init blog

然后验证是否初始化成功，执行如下命令，开启本地Hexo服务器：

hexo s

此时在浏览器地址栏输入localhost:4000，应该能看到如下页面：

如果出错，有可能是因为4000端口被占用了，使用如下命令修改端口再试一下：

hexo s -p 5000

3.使用hexo写博客

3.1为什么使用hexo

本来我是只用GitHub提供的Pages功能的，这样也能写博客，并且在公网上能看见。但这样就会出现一些问题：博客目录需要手动更新，每次写完一篇文章就要去readme.md里加一条新地址，这样倒是也能接受，但关键还有很多大问题，如：文章多了没法分页，博客没法设置标签，评论、点赞、流量统计、博客模板之类的功能肯定是都不可能有了。这样一些配合GitHub的博客框架就应运而生了，hexo就是其中之一。我目前刚刚接触这一块，看了一些牛人的博客，有已经从hexo迁走的了，也许我以后也会尝试新的框架，现在先搞好hexo。
